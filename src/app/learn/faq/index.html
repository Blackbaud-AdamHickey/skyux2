Frequently asked questions

This section provides answers to frequently asked questions about SKY UX 2. ... These questions and answers are taken from the 1bb-skyux-apps Slack channel. Should we link to that from here?

What is the recommended best practice for getting the latest

The command to get the latest is npm install -g @blackbaud/skyux-cli@latest. The CLI notifies users when a new version is released. When users run skyux serve, a message appears in the command prompt if a new version is available.

What is the timeline for SKY UX 2 to be production-ready?

SKY UX components are production-ready now. We are still in beta because not every component has been moved over and because we still need to create quite a bit of documentation.

What is the timeline to be able to deploy SKY SPAs for use in production?

The SKY UX Host, Builder, and CLI pieces can be used to build a production app. Blue Steel is doing this now with a release target of February. It just requires some manual steps and patience with some suboptimal workflow while we automate more and add more user-friendly features.

Will deployment of SKY UX SPAs be automated through https://github.com/blackbaud/skyux-deploy?

Yes. Currently, any merge to your master ranch triggers a release to production. We plan to change that to state a release with an endpoint or a user interface where you can flag staged releases as production releases.

Since the app repository exists solely for UI components, do you need a separate repo if you need a backend deployed to perform business logic or some other action?
